## Загальні відомості про Nginx

`Nginx` — це вебсервер і `reverse proxy`, який дуже часто стоїть перед backend-додатком у production. Його головна задача — приймати HTTP-запити від користувачів і вирішувати, що з ними робити далі: віддати статичний файл, перенаправити запит у backend, або обробити SSL-з'єднання.

У типовій архітектурі користувач не звертається напряму до backend-сервера. Спочатку запит потрапляє в nginx, а вже nginx передає його далі у backend, наприклад у `Django` або `FastAPI`. Це називається `reverse proxy`. Такий підхід дає кілька переваг: backend не потрібно відкривати напряму в інтернет, `nginx` може балансувати навантаження, а також значно краще працює з великою кількістю одночасних підключень.

Одна з дуже важливих задач nginx — це `SSL termination`. Саме nginx працює з HTTPS-сертифікатами: приймає захищене з'єднання від браузера, розшифровує його і вже далі передає запит у `backend` по звичайному `HTTP` всередині `Docker network`. Завдяки цьому backend не потрібно окремо налаштовувати для SSL.

Ще одна сильна сторона `nginx` — це робота зі статичними файлами. Наприклад, `CSS`, `JavaScript`, зображення або зібрані Django static files (manage.py collectstatic) nginx може віддавати напряму з файлової системи без участі backend. Це значно швидше, ніж якщо кожен файл проходить через Python-застосунок.

Також nginx дозволяє дуже зручно робити маршрутизацію. Наприклад:

/ -> backend
/adminer/ -> окремий контейнер з Adminer
/static/ -> статичні файли

Тобто один домен може обслуговувати відразу кілька сервісів.

Для production `nginx` часто використовують і як базовий захист. Наприклад, через `rate limiting` можна обмежити кількість запитів від одного клієнта, щоб захиститися від надто агресивного трафіку або примітивних атак. Якщо один IP починає надсилати занадто багато запитів за секунду, `nginx` просто починає відхиляти зайві запити ще до того, як вони дійдуть до backend.

`Nginx` працює дуже ефективно: він використовує неблокуючу модель обробки запитів і споживає значно менше ресурсів, ніж багато класичних серверів.

Якщо сказати дуже просто, то в production ролі виглядають так:

Nginx приймає запит.
Backend виконує логіку.
Database зберігає дані.

Саме тому nginx майже завжди є частиною сучасної backend-архітектури 🚀
